# Agentic RAG 智能知识问答系统 — 需求快照

## 1. 产品目标与范围

构建面向企业与个人知识库的 Agentic RAG 智能知识问答系统。系统支持用户上传、管理、检索文档，并通过可追溯、可解释的智能问答获得基于文档依据的答案。

首期范围包含：用户登录与权限控制、文档上传与处理、知识库创建与文档绑定、基础 RAG 问答、引用来源展示、多轮会话、简单 Agent 工具调用、管理员查看文档处理状态。

首期明确不包含：多 Agent 协作、自动网页爬取、复杂工作流平台、私有化模型训练、多模态图片/视频理解、企业级审计合规模块、高复杂权限体系。

非阻塞默认：按 Web 应用 + 后端 API + 异步文档处理 Worker + 可替换模型/存储适配器的项目边界建模；具体前端框架、后端框架、向量库和模型供应商保持可替换。

## 2. 用户、参与者与消费者

- 普通用户：上传文档、创建或使用有权限的知识库、发起问答、查看引用来源与历史会话。
- 管理员：管理用户、知识库、文档、系统配置，并查看文档处理状态。
- 系统 Agent：作为系统内部执行者，负责意图判断、检索规划、工具调用、多轮上下文整合和答案生成。
- 外部模型/存储/检索能力：大模型、Embedding、重排序模型、向量数据库、对象存储、缓存、任务队列等可作为基础设施或外部服务接入。

## 3. 主要产品场景

- 用户登录后访问授权知识库和文档。
- 用户上传 PDF、DOCX、TXT、MD、HTML 等文档，系统异步解析、清洗、切分、向量化并更新处理状态。
- 用户创建知识库、绑定文档、按知识库/标签/上传时间管理文档。
- 用户指定知识库提问，系统执行语义检索、关键词检索、混合检索与重排序，生成带引用来源的回答。
- 系统 Agent 针对摘要、对比、解释、信息提取等问题自动规划多轮检索、拆解子问题并调用工具。
- 用户在历史会话中连续追问，系统结合上下文补全省略问题。
- 管理员查看文档处理状态和失败原因，执行删除、重新解析、元数据维护等操作。

## 4. 对外暴露接口

- Web 用户端：知识库管理、文档上传、问答会话、引用查看、历史会话查看与删除。
- Web 管理端：用户/知识库/文档管理、系统配置、文档处理状态查看。
- 后端 REST API：认证、用户、知识库、文档、会话、问答、管理员操作接口。
- 异步任务入口：文档解析、切分、向量化、失败重试、重新解析。
- 模型与工具适配接口：LLM 生成、Embedding、重排序、文档解析、引用定位、摘要、表格解析、全文/向量检索。

## 5. 能力分组

- auth：注册、登录、退出、角色识别、API 鉴权。
- access-control：用户数据隔离、知识库与文档权限校验、越权防护。
- knowledge-base：知识库创建、描述、权限范围、文档绑定、分类管理。
- document-ingestion：上传、解析、清洗、切分、向量化、重新解析、处理状态。
- retrieval：向量检索、关键词检索、混合检索、重排序、召回质量控制。
- agent-orchestration：意图识别、检索规划、子问题拆解、工具调用、多轮检索、答案依据校验。
- conversation：会话创建、历史查看与删除、多轮上下文记忆、追问补全。
- answer-generation：结构化回答、摘要/对比/列表/表格输出、低置信度拒答、文档说明与系统推断区分。
- citation：引用来源定位、文档名、段落位置或页码展示、检索结果摘要保存。
- admin：用户、文档、知识库、配置和处理状态管理。

## 6. 架构可见职责

- 前端负责用户与管理员交互、上传入口、问答界面、引用展示和状态可视化。
- API 接入层负责认证、鉴权、限流、请求路由和访问日志。
- 应用用例层负责把接口请求编排为用户、知识库、文档、问答、会话和管理员用例。
- Agent/RAG 编排层负责意图判断、检索策略、多工具调用、答案合成与置信度控制。
- 文档智能服务负责解析、清洗、切分、Embedding、索引、重排序、引用定位和摘要/表格解析。
- 领域服务负责用户权限、知识库、文档元数据、会话记录、检索归因等业务规则。
- 适配器层隔离模型、向量库、全文检索、对象存储、解析器和缓存实现。
- 异步任务层负责耗时文档处理、状态更新、失败重试和后台处理可观测性。

## 7. 数据、状态与产物

- 用户、角色、权限、知识库、文档元数据、标签、文档绑定关系、任务状态、会话消息、答案、引用来源、检索结果摘要等结构化数据。
- 原始上传文件和可选解析中间产物。
- 文档分块、分块元数据、页码/段落定位、Embedding 向量、向量索引、关键词全文索引。
- Redis 或等价缓存中的会话上下文、热点查询结果、任务进度短期状态。
- 处理日志、失败原因、追踪指标和运维日志。

## 8. 外部系统与生态

- LLM 生成模型服务：用于答案生成、摘要、解释、对比、信息提取。
- Embedding 模型服务：用于文档分块与查询向量化。
- 重排序模型服务：用于召回结果排序优化。
- 文档解析库/服务：Docling、Unstructured、PyMuPDF 等可替换实现。
- 向量数据库/索引：Milvus、Qdrant、pgvector 等可替换实现。
- 关系数据库：PostgreSQL/MySQL 等。
- 对象存储：本地存储、MinIO、OSS 等。
- 缓存与异步任务：Redis、Celery/RQ/Dramatiq 等。

## 9. 运行、部署与运维边界

- 浏览器前端与后端 API 服务分离。
- 文档上传请求与文档解析/切分/向量化 Worker 分离，避免阻塞普通问答请求。
- 模型、向量库、对象存储、缓存、关系数据库通过适配器接入，便于替换。
- 管理员可观测文档处理状态与失败原因。
- 配置与密钥管理、日志指标追踪、备份恢复、发布回滚属于基础运行能力。

## 10. 架构约束

- 回答必须基于检索到的文档内容，并提供可追溯引用。
- 未检索到充分依据时必须明确说明，避免幻觉回答。
- 权限隔离是核心约束：用户只能访问有权限的知识库和文档。
- 文档解析与向量化采用异步任务处理。
- 检索、重排序、生成、引用管理、Agent 工具调用需要解耦封装。
- 支持后续切换模型、Embedding、向量数据库和新增工具。
- 普通问答响应时间需要可接受；热点查询和会话上下文可缓存。

## 11. 流程候选

- 用户登录与授权访问流程：说明认证、角色识别、权限守卫如何保护知识库与文档访问。
- 文档上传入库流程：说明上传、对象存储、异步解析、切分、Embedding、索引和状态更新。
- 指定知识库 Agentic RAG 问答流程：说明前端提问、会话上下文、意图识别、检索规划、混合检索、重排序、引用定位、LLM 生成与拒答策略。
- 多轮追问流程：说明历史上下文读取、问题补全和会话持久化。
- 管理员查看文档处理状态流程：说明管理员入口、权限校验、任务状态读取与失败原因展示。
